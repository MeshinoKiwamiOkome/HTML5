① 基本的な書式と各部の名称

ポイント
CSSは セレクタ・プロパティ・値 の3つで構成されます。
セレクタとプロパティ・値のペアをまとめて ルールセット（スタイルルール） と呼びます。
h1 セレクタ（どの要素に適用するか）
{ 宣言ブロック 開始
    color プロパティ（何を変えるか） : red 値（どう変えるか） ; ← 宣言（プロパティ + 値）
    font-size: 24px;
} 宣言ブロック 終了 
/* ルールセット（スタイルルール）の基本構造 */
セレクタ {
    プロパティ: 値;
    プロパティ: 値;
}

/* 実際の例 */
h1 {
    color: red;
    font-size: 24px;
}

/* 複数のセレクタに同じスタイルを適用（グループ化） */
h1, h2, h3 {
    color: #2b6cb0;
}
⚠️ 試験重要ポイント①：空白文字の扱い

CSSの書式には自由に空白文字（スペース・タブ・改行）を入れられますが、 セレクタだけは例外です。

セレクタ内の空白は「子孫セレクタ」として解釈されるため、 意図しない意味になります。

✅ プロパティ・値・コロン・セミコロン周辺は自由に空白を入れられる
color : red ;　← 文法的に正しい（コロン前後の空白OK）

❌ セレクタに意図しない空白を入れると別の意味になる
h 1 { color: red; }　← h 要素の子孫の 1 という意味になる（不正）
h1 { color: red; }　← 正しい（h1要素を指定）

✅ セレクタのグループ化（カンマ区切り）の前後の空白はOK
h1 , h2 { color: red; }　← 正しい
用語説明例
セレクタスタイルを適用する対象の指定h1 / .class / #id
プロパティ変更するスタイルの種類color / font-size
値プロパティに設定する内容red / 24px
宣言プロパティと値のペアcolor: red
宣言ブロック{ } で囲まれた宣言の集まり{ color: red; }
ルールセットセレクタ＋宣言ブロック全体h1 { color: red; }

② link要素でHTMLに組み込む（外部スタイルシート）

ポイント②：link要素に type="text/css" の指定は不要
HTML5以降では type="text/css" はデフォルト値のため省略できます。 
<!-- ❌ HTML4時代の書き方（type属性が必要だった） -->
<link rel="stylesheet"
      type="text/css"
      href="style.css">

<!-- ✅ HTML5以降の正しい書き方（type属性は省略可） -->
<link rel="stylesheet"
      href="style.css">

<!-- ✅ media属性でメディアクエリを指定できる -->
<link rel="stylesheet"
      href="style.css"
      media="screen">

<link rel="stylesheet"
      href="print.css"
      media="print">

<!-- ✅ media属性にメディアクエリを指定（ポイント③） -->
<link rel="stylesheet"
      href="mobile.css"
      media="screen and (max-width: 600px)">
⚠️ 試験重要ポイント③：media属性にはメディアクエリが指定可能
link 要素の media 属性には screen / print などのメディアタイプだけでなく、 screen and (max-width: 600px) のような メディアクエリを指定できます。
属性説明備考
relリソースの関係性"stylesheet" を指定（必須）
hrefCSSファイルのURL必須
typeMIMEタイプHTML5以降は省略可（デフォルト: text/css）
media適用するメディアメディアクエリも指定可
✅ 外部スタイルシートのメリット
・複数のHTMLファイルに同じCSSを適用できる
・CSSファイルがブラウザにキャッシュされ、ページ読み込みが高速化
・HTMLとCSSの役割を分離できる（保守性向上）

③ style要素でHTMLに組み込む（内部スタイルシート）

ポイント②：style要素に type="text/css" の指定は不要
style 要素も同様に、HTML5以降では type="text/css" は省略できます。 
<!-- ❌ HTML4時代の書き方 -->
<style type="text/css">
  h1 { color: red; }
</style>

<!-- ✅ HTML5以降の正しい書き方（type属性は省略可） -->
<style>
  h1 { color: red; }
</style>

<!-- ✅ media属性でメディアクエリを指定できる（ポイント③） -->
<style media="screen and (max-width: 600px)">
  body { font-size: 0.85em; }
</style>

<!-- style要素は通常 head 要素内に記述する -->
<head>
  <style>
    body { font-family: sans-serif; }
    h1   { color: #2b6cb0; }
  </style>
</head>
属性説明備考
typeMIMEタイプHTML5以降は省略可（デフォルト: text/css）
media適用するメディアメディアクエリも指定可

④ style属性でHTMLに組み込む（インラインスタイル）

ポイント
・style 属性はHTML要素に直接スタイルを指定します。
・セレクタは不要で、プロパティと値のペアを直接記述します。
・最も優先度が高いスタイル指定方法です（!important を除く）。 
<!-- style属性の書き方：セレクタは不要 -->
<p style="color: red; font-size: 18px;">
  赤色・18pxのテキスト
</p>

<h1 style="color: #2b6cb0; border-bottom: 2px solid #4299e1;">
  見出し
</h1>

style属性で紫色・18pxを指定したテキスト

style属性で色と下線を指定した見出し

⚠️ インラインスタイルの注意点
・media 属性は使えません（メディアクエリ不可）
・セレクタが不要なため、複数要素への一括適用はできません
・HTMLとCSSが混在するため保守性が下がります
・JavaScriptで動的にスタイルを変更する場合などに限定して使うのが推奨です
組み込み方法場所type属性media属性優先度
link要素（外部）head内省略可✅ メディアクエリ可低
style要素（内部）head内省略可✅ メディアクエリ可中
style属性（インライン）各要素なし❌ 不可高

⑤ @importでCSSに組み込む

ポイント③：@importにはメディアクエリが指定可能
・@import はCSSファイルの中から別のCSSファイルを読み込む方法です。
・CSSファイルの先頭に記述する必要があります（他のルールより前）。
・@import にもメディアクエリを指定できます。 
/* ===== style.css の中での @import ===== */

/* 基本的な書き方 */
@import "reset.css";
@import url("typography.css");

/* メディアタイプを指定 */
@import "print.css" print;

/* メディアクエリを指定（ポイント③） */
@import "mobile.css" screen and (max-width: 600px);
@import "dark.css"   screen and (prefers-color-scheme: dark);

/* ❌ @importはCSSの先頭に書く（他のルールの後はNG） */
body { color: black; }
@import "other.css";  /* ← 無視される */
⚠️ @importの注意点
・必ずCSSファイルの先頭（他のルールセットより前）に記述します
・他のルールの後に書いた @import は無視されます
・@import は読み込みが直列になるため、 link 要素による並列読み込みよりパフォーマンスが低下する場合があります
・url() は省略可能（文字列のみでもOK）
書き方説明
@import "file.css";文字列で指定（url()省略可）
@import url("file.css");url()で指定
@import "file.css" print;メディアタイプを指定
@import "file.css" screen and (max-width: 600px);メディアクエリを指定

⑥ ボックスの構造（ボックスモデル）

ポイント
HTMLの各要素はすべてボックス（箱）として扱われます。
ボックスは外側から margin → border → padding → content の4層で構成されます。
margin（外側の余白）
border（枠線）
padding（内側の余白）
content
（コンテンツ領域） 
/* ボックスモデルの各部分 */
.box {
    /* content：コンテンツ領域のサイズ */
    width:  300px;
    height: 200px;

    /* padding：内側の余白（content と border の間） */
    padding:        20px;          /* 上下左右すべて */
    padding-top:    10px;
    padding-right:  20px;
    padding-bottom: 10px;
    padding-left:   20px;

    /* border：枠線 */
    border:       2px solid #4299e1;
    border-width: 2px;
    border-style: solid;
    border-color: #4299e1;

    /* margin：外側の余白（要素と要素の間隔） */
    margin:        16px;          /* 上下左右すべて */
    margin-top:    16px;
    margin-right:  auto;          /* 中央揃えに使用 */
    margin-bottom: 16px;
    margin-left:   auto;
}

/* box-sizing：ボックスサイズの計算方法 */
.box-content {
    box-sizing: content-box;  /* デフォルト：width = content のみ */
}
.box-border {
    box-sizing: border-box;   /* width = content + padding + border */
}
box-sizingwidthが示す範囲用途
content-box contentのみ
（実際の幅 = width + padding + border）		 デフォルト値
border-box content + padding + border
（実際の幅 = width のまま）		 レイアウト計算が直感的。
* { box-sizing: border-box; } が一般的
✅ margin の特殊な動作
・margin: auto を左右に指定するとブロック要素を水平中央揃えにできます
・上下に隣接するブロック要素のmarginはマージンの相殺（大きい方だけが適用）が起きます
・margin は背景色が適用されません（透明な余白）

⑦ 長さを表す単位

ポイント
CSSの単位は絶対単位と相対単位に分類されます。
レスポンシブデザインでは相対単位の活用が重要です。

絶対単位

単位名称説明
pxピクセル画面の物理ピクセル（最も一般的な絶対単位）
ptポイント1pt = 1/72インチ。印刷向け
pcパイカ1pc = 12pt。印刷向け
cmセンチメートル実際の長さ。印刷向け
mmミリメートル実際の長さ。印刷向け
inインチ1in = 2.54cm = 96px
Qクォーター・ミリメートル1Q = 0.25mm。印刷向け

相対単位

単位名称基準説明
em エム 親要素のフォントサイズ 親が16pxなら 1em = 16px。ネストで累積
rem ルートエム ルート要素（html）のフォントサイズ 累積しない。レスポンシブに最適
% パーセント 親要素の同プロパティの値 幅・高さ・フォントサイズなどに使用
vw ビューポート幅 ビューポートの幅 1vw = ビューポート幅の1%
vh ビューポート高さ ビューポートの高さ 1vh = ビューポート高さの1%
vmin ビューポート最小辺 vw / vh の小さい方 1vmin = vw・vhのうち小さい方の1%
vmax ビューポート最大辺 vw / vh の大きい方 1vmax = vw・vhのうち大きい方の1%
ch 文字幅 「0」の文字幅 テキスト幅の制御に使用
ex エックスハイト 小文字「x」の高さ フォントの高さ基準 
/* 単位の使い分け例 */
html {
    font-size: 16px;       /* ルートフォントサイズ */
}
body {
    font-size: 1rem;       /* = 16px（ルート基準） */
    max-width: 960px;
    padding:   0 1.5rem;   /* = 0 24px */
}
h1 {
    font-size: 2rem;        /* = 32px（ルート基準・累積なし） */
}
.hero {
    height:    100vh;       /* ビューポートの高さいっぱい */
    font-size: 5vw;         /* ビューポート幅に応じたサイズ */
}
.container {
    width:     90%;         /* 親要素の90% */
    margin:    0 auto;      /* 中央揃え */
}
⚠️ em と rem の違い（試験頻出）

em：親要素のフォントサイズを基準とするため、 ネストが深くなると累積して大きくなります
例：親が 16px → 子が 2em = 32px → 孫が 2em = 64px

rem：常にルート要素（html）のフォントサイズを基準とするため 累積しません
例：html が 16px → どこで使っても 2rem = 32px
