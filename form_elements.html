📝 HTML5 フォーム要素 学習サンプル

form / input / textarea / button / select / option / optgroup / meter / progress / datalist / output / label / fieldset / legend の役割と使い方を学びましょう。

① form要素

ポイント
・form 要素はフォームコントロールをまとめ、 サーバーへのデータ送信を定義するブロック要素です。
・action 属性で送信先URL、method 属性で送信方法を指定します。 
<!-- 基本的なform要素 -->
<form action="/submit"
      method="post"
      enctype="multipart/form-data"
      novalidate>
  <!-- フォームコントロール -->
</form>
属性説明値の例
action送信先URL"/submit"
method送信方法get / post
enctypeエンコード形式（ファイル送信時に必要）multipart/form-data
novalidateブラウザのバリデーションを無効化（論理属性）—
autocompleteフォーム全体のオートコンプリートon / off
target送信結果の表示先_blank / _self

③ autocomplete属性（HTML5.1以降で大幅拡張）

ポイント②：HTML5.1以降ではautocomplete属性が大幅に拡張されている
・HTML5では on / off のみでしたが、 HTML5.1以降では詳細なトークン値が追加されました。
・ブラウザが適切な候補を自動補完できるようになります。
・パスワードマネージャーとの連携にも重要です。 
<!-- HTML5：on / off のみ -->
<input type="text" autocomplete="on">
<input type="text" autocomplete="off">

<!-- HTML5.1以降：詳細なトークン値 -->
<input type="text"     autocomplete="name">
<input type="text"     autocomplete="given-name">
<input type="text"     autocomplete="family-name">
<input type="email"    autocomplete="email">
<input type="tel"      autocomplete="tel">
<input type="text"     autocomplete="postal-code">
<input type="text"     autocomplete="address-line1">
<input type="text"     autocomplete="organization">
<input type="password" autocomplete="current-password">
<input type="password" autocomplete="new-password">
<input type="text"     autocomplete="one-time-code">
<input type="text"     autocomplete="cc-number">
<input type="text"     autocomplete="bday">
<input type="text"     autocomplete="sex">
<input type="url"      autocomplete="url">
<input type="text"     autocomplete="username">
カテゴリ主なトークン値
名前name / given-name / family-name / nickname
連絡先email / tel / url
住所address-line1 / address-line2 / postal-code / country
認証username / current-password / new-password / one-time-code
クレジットカードcc-number / cc-name / cc-exp
その他organization / bday / sex / language

④ textarea要素

<!-- 複数行テキスト入力 -->
<textarea name="message"
          rows="5"
          cols="40"
          placeholder="メッセージを入力してください"
          maxlength="500"
          required>
</textarea>
⚠️ textarea要素の注意点
・textarea は空要素ではありません（開始タグと終了タグが必要）
・初期値はタグ内のテキストで指定します（value 属性はありません）
・rows / cols 属性でサイズを指定しますが、CSSでの指定が推奨です

⑤ button要素

<!-- type="submit"：フォームを送信（デフォルト） -->
<button type="submit">送信する</button>

<!-- type="reset"：フォームをリセット -->
<button type="reset">リセット</button>

<!-- type="button"：JavaScriptで使う汎用ボタン -->
<button type="button"
        onclick="alert('クリックされました')">
  クリック
</button>

<!-- disabled属性：無効化 -->
<button type="submit" disabled>送信（無効）</button>
⚠️ button要素のtype属性
・type 属性を省略するとデフォルトは submit になります
・フォーム外で使う場合は必ず type="button" を明示しましょう
・input type="submit" と異なり、内部にHTML要素を含められます（アイコンなど）

⑥ select要素 / option要素 / optgroup要素

<!-- 基本的なselect -->
<select name="prefecture">
  <option value="">選択してください</option>
  <option value="tokyo">東京都</option>
  <option value="osaka" selected>大阪府</option>
  <option value="okayama">岡山県</option>

<!-- optgroup：選択肢をグループ化 -->
<select name="language">
  <optgroup label="フロントエンド">

    <option value="php">PHP</option>

  </optgroup>

<!-- multiple：複数選択 -->
<select name="skills" multiple size="4">
  <option value="html">HTML</option>
  <option value="css">CSS</option>
  <option value="js">JavaScript</option>
  <option value="php">PHP</option>
</select>

⑦ datalist要素

ポイント③：datalist要素はinput要素に関連付けてサジェスト機能を追加する
・datalist 要素は input 要素の list 属性で関連付けます。
・select と異なり、自由入力もできる（候補から選ぶことも、直接入力することもできる）。
・datalist 自体は画面に表示されません。 
<!-- inputのlist属性とdatalistのidを一致させる -->
<label for="browser">ブラウザ：</label>
<input type="text"
       id="browser"
       name="browser"
       list="browser-list"
       placeholder="入力または選択">

<datalist id="browser-list">
  <option value="Chrome">
  <option value="Firefox">
  <option value="Safari">
  <option value="Edge">
  <option value="Opera">
</datalist>
⚠️ datalist と select の違い
・datalist：候補を提示しつつ自由入力も可能（サジェスト）
・select：候補の中からのみ選択（自由入力不可）

・datalist の id と input の list 属性値を必ず一致させます

⑧ meter要素 / progress要素

<!-- meter：既知の範囲内のスカラー値（ゲージ） -->
<p>ディスク使用量：
  <meter value="7"
         min="0"
         max="10"
         low="3"
         high="8"
         optimum="2">
    7/10
  </meter>
</p>

<!-- progress：タスクの進捗状況 -->
<p>ダウンロード進捗：
  <progress value="60" max="100">60%</progress>
</p>

<!-- progress（不確定）：value属性なしで不確定な進捗 -->
<p>読み込み中：
  <progress>読み込み中…</progress>
</p>

ディスク使用量：7/10 7/10GB

ダウンロード進捗：60% 60%

読み込み中：読み込み中…（不確定）

⑨ output要素

ポイント
・output 要素はフォームの計算結果や出力値を表示するための要素です。
・for 属性で関連する入力要素のIDを指定します。 
<form oninput="result.value = parseInt(a.value) + parseInt(b.value)">
  <input type="number" id="a" value="10">
  +
  <input type="number" id="b" value="20">
  =
  <output name="result" for="a b">30</output>
</form>

▼ 数値の合計をリアルタイム表示

　+　 　=　 30

⑩ label要素 / fieldset要素 / legend要素

<!-- label：for属性でinputと関連付け -->
<label for="username">ユーザー名：</label>
<input type="text" id="username" name="username">

<!-- label：inputを内包する書き方（for属性不要） -->
<label>
  メールアドレス：
  <input type="email" name="email">
</label>

<!-- fieldset + legend：関連するコントロールをグループ化 -->
<fieldset>
  <legend>個人情報</legend>
  <label for="fname">名前：</label>
  <input type="text" id="fname">
  <br>
  <label for="femail">メール：</label>
  <input type="email" id="femail">
</fieldset>

<!-- disabled属性でfieldset全体を無効化 -->
<fieldset disabled>
  <legend>支払い情報（現在無効）</legend>
  <input type="text" placeholder="カード番号">
</fieldset>
個人情報
✅ labelのアクセシビリティ効果
・label をクリックすると関連する input にフォーカスが移ります
・スクリーンリーダーがフォームコントロールの目的を読み上げます
・for 属性の値と input の id 属性の値を必ず一致させます
