📋 HTML5 リスト要素 学習サンプル

ul・ol・li・dl・dt・dd の役割と使い方を学びましょう。

① リスト要素の全体像

ul（順不同リスト）

・順序に意味がないリスト
・箇条書き（●）で表示
・子要素は li のみ
・内容は空でもOK

ol（順序付きリスト）

・順序に意味があるリスト
・番号付き（1.2.3.）で表示
・子要素は li のみ
・内容は空でもOK

dl（説明リスト）

・用語と説明のペア
・dt（用語）+ dd（説明）
・用語定義には dfn も使う
・内容は空でもOK
⚠️ 試験重要ポイント①：ul・ol・dl の内容は空でもOK
HTMLの仕様上、ul / ol / dl 要素は 子要素が0個（空）でも文法的に正しいです。
JavaScriptで動的にリストを生成する場合などに空のリスト要素を使うことがあります。

✅ <ul></ul>　← 文法的に正しい
✅ <ol></ol>　← 文法的に正しい
✅ <dl></dl>　← 文法的に正しい

② ul要素（順不同リスト）

ポイント
・ul（Unordered List）は順序に意味がない箇条書きリストです。
・直接の子要素は li 要素のみです（div や p は直接置けません）。
・ナビゲーションメニュー・機能一覧・タグ一覧などに使います。 
<!-- 基本的な使い方 -->
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

<!-- ネスト（入れ子）-->
<ul>
  <li>フロントエンド
    <ul>
      <li>HTML</li>
      <li>CSS</li>
    </ul>
  </li>
  <li>バックエンド
    <ul>
      <li>PHP</li>
      <li>Python</li>
    </ul>
  </li>
</ul>

<!-- 空のulも文法的に正しい -->
<ul id="dynamic-list"></ul>
<!-- JavaScriptで後からliを追加する場合など -->

▼ 基本表示

  • HTML
  • CSS
  • JavaScript

▼ ネスト表示

  • フロントエンド
    • HTML
    • CSS
  • バックエンド
    • PHP
    • Python
⚠️ ul要素の注意点
・ul の直接の子要素は li のみです
❌ <ul><p>テキスト</p></ul>　← 不正
✅ <ul><li>テキスト</li></ul>　← 正しい

・ナビゲーションメニューは nav 要素と組み合わせて使います

③ ol要素（順序付きリスト）

ポイント
・ol（Ordered List）は順序に意味がある番号付きリストです。
・手順・ランキング・レシピの工程など、順番が重要なコンテンツに使います。
・start 属性で開始番号、reversed 属性で逆順を指定できます。 
<!-- 基本的な使い方 -->
<ol>
  <li>HTMLを学ぶ</li>
  <li>CSSを学ぶ</li>
  <li>JavaScriptを学ぶ</li>
</ol>

<!-- start属性：開始番号を指定 -->
<ol start="3">
  <li>3番目から始まる</li>
  <li>4番目</li>
  <li>5番目</li>
</ol>

<!-- reversed属性：逆順（論理属性） -->
<ol reversed>
  <li>3位：山田太郎</li>
  <li>2位：鈴木花子</li>
  <li>1位：田中一郎</li>
</ol>

<!-- type属性：番号の種類 -->
<ol type="A">  <!-- A, B, C... -->
  <li>選択肢A</li>
  <li>選択肢B</li>
</ol>

<!-- li要素のvalue属性：個別の番号を指定 -->
<ol>
  <li value="1">第1章</li>
  <li value="5">第5章（2〜4を飛ばす）</li>
  <li>第6章（自動で続く）</li>
</ol>

▼ 基本表示

  1. HTMLを学ぶ
  2. CSSを学ぶ
  3. JavaScriptを学ぶ

▼ reversed属性（逆順）

  1. 3位：山田太郎
  2. 2位：鈴木花子
  3. 1位：田中一郎

▼ start="3"（3から開始）

  1. 3番目から始まる
  2. 4番目
  3. 5番目

▼ type="A"（アルファベット）

  1. 選択肢A
  2. 選択肢B
  3. 選択肢C
⚠️ 試験重要ポイント②：reversed属性
・reversed 属性は論理属性（値なしで指定）です
・リストアイテムの表示順序は変わりません（HTMLに書いた順のまま）
・変わるのは番号の付き方だけです（大きい番号から小さい番号へ）

例：3つのliがある場合 → 3, 2, 1 の順に番号が付く
属性対象説明
reversedol番号を逆順にする（論理属性）
startol開始番号を整数で指定
typeol番号の種類（1 / a / A / i / I）
valueliそのli要素の番号を個別に指定

④ li要素（リストアイテム）

ポイント
・li（List Item）は ul または ol の子要素です。
・li 要素の中にはブロック要素・インライン要素を含めることができます。
・li 要素は ul / ol の外に単独で置くことはできません。 
<!-- liの中にブロック要素を含める -->
<ul>
  <li>
    <h3>HTML5の特徴</h3>
    <p>セマンティックな要素が追加されました。</p>
  </li>
  <li>
    <h3>CSS3の特徴</h3>
    <p>アニメーションやグラデーションが使えます。</p>
  </li>
</ul>

<!-- ナビゲーションメニューでの使用例 -->
<nav>
  <ul>
    <li><a href="/">ホーム</a></li>
    <li><a href="/about">会社概要</a></li>
    <li><a href="/contact">お問い合わせ</a></li>
  </ul>
</nav>
⚠️ li要素の注意点
・li は必ず ul または ol の子要素として使います
❌ <div><li>項目</li></div>　← 不正（ulもolも親にない）
✅ <ul><li>項目</li></ul>　← 正しい

⑤ dl要素 / dt要素 / dd要素（説明リスト）

ポイント
・dl（Description List）は用語と説明のペアを表すリストです。
・dt（Description Term）：用語・名前・質問など
・dd（Description Details）：用語の説明・定義・回答など
・1つの dt に複数の dd を対応させることができます。
・複数の dt を1つの dd に対応させることもできます。 
<!-- 基本的な使い方 -->
<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language の略。ウェブページの構造を定義する言語。</dd>

  <dt>CSS</dt>
  <dd>Cascading Style Sheets の略。ウェブページの見た目を定義する言語。</dd>
</dl>

<!-- 1つのdtに複数のdd -->
<dl>
  <dt>プログラミング言語</dt>
  <dd>JavaScript</dd>
  <dd>Python</dd>
  <dd>PHP</dd>
</dl>

<!-- 複数のdtに1つのdd -->
<dl>
  <dt>HTML</dt>
  <dt>HyperText Markup Language</dt>
  <dd>ウェブページの構造を定義するマークアップ言語。</dd>
</dl>

<!-- FAQへの応用 -->
<dl>
  <dt>試験の難易度はどのくらいですか？</dt>
  <dd>HTML・CSSの基礎知識があれば挑戦できるレベルです。</dd>

  <dt>受験資格はありますか？</dt>
  <dd>年齢・学歴・職歴などの制限はありません。</dd>
</dl>

▼ 基本表示

HTML
HyperText Markup Language の略。ウェブページの構造を定義する言語。
CSS
Cascading Style Sheets の略。ウェブページの見た目を定義する言語。

▼ 1つのdtに複数のdd

プログラミング言語
JavaScript
Python
PHP

⑥ dl要素 と dfn要素の組み合わせ

ポイント③：dl要素で用語の定義をする場合、dfn要素も必要になる
・dl 要素で用語の定義を行う場合は、 dt 要素内に dfn 要素を使って定義語を明示します。
・dfn 要素はその文書内での用語の定義箇所を示します。
・単なる用語と説明のペア（FAQなど）には dfn は不要です。 
<!-- 用語の定義にdlを使う場合はdfnも使う -->
<dl>
  <dt>
    <dfn>セマンティクス</dfn>
  </dt>
  <dd>
    HTMLタグに意味を持たせることで、
    文書構造を人間とコンピュータの両方に
    わかりやすくする考え方。
  </dd>

  <dt>
    <dfn><abbr title="HyperText Markup Language">HTML</abbr></dfn>
  </dt>
  <dd>
    ウェブページの構造を定義するマークアップ言語。
    W3CとWHATWGが仕様を策定している。
  </dd>
</dl>

<!-- FAQなど「定義」でない場合はdfn不要 -->
<dl>
  <dt>営業時間は何時ですか？</dt>  <!-- 定義ではないのでdfn不要 -->
  <dd>平日9時〜18時です。</dd>
</dl>

▼ dfnを使った用語定義の表示

セマンティクス
HTMLタグに意味を持たせることで、文書構造を人間とコンピュータの両方にわかりやすくする考え方。
HTML
ウェブページの構造を定義するマークアップ言語。W3CとWHATWGが仕様を策定している。
✅ dlの使い分けまとめ
・用語の定義（glossary）→ dl + dt（dfnあり）+ dd
・FAQ・Q&A → dl + dt（dfnなし）+ dd
・メタデータのペア（著者・公開日など）→ dl + dt + dd
用途dfnの要否例
用語の定義（glossary） ✅ 必要 技術用語集・辞書
FAQ・Q&A ❌ 不要 よくある質問ページ
メタデータのペア ❌ 不要 著者名・公開日・カテゴリ

⑦ ul / ol / dl の使い分け

要素使う場面具体例
ul 順序に意味がないリスト ナビメニュー・機能一覧・タグ一覧・買い物リスト
ol 順序に意味があるリスト 手順・ランキング・レシピ・目次
dl 用語と説明のペア 用語集・FAQ・メタデータ（著者・日付など）
⚠️ よくある誤用
❌ インデント（字下げ）目的で ul を使う → CSSの padding / margin で対応
❌ 番号を付けたいだけで ol を使う（順序に意味がない場合）→ ul + CSS
❌ 見た目のリスト表示のために dl を使う → 用語と説明のペアでない場合は不適切

📌 リスト要素 まとめ一覧

要素役割重要ポイント
ul 順不同リスト 順序に意味がない箇条書き。直接の子はliのみ。空でもOK
ol 順序付きリスト 順序に意味がある番号付き。reversed / start / type属性あり。空でもOK
li リストアイテム ul / ol の子要素。単独使用不可。value属性で番号を個別指定可（ol内）
dl 説明リスト 用語と説明のペア。FAQ・用語集・メタデータに使用。空でもOK
dt 説明リストの用語 dl内で使用。用語定義の場合はdfn要素を含める
dd 説明リストの説明 dl内で使用。1つのdtに複数のddを対応させることができる
