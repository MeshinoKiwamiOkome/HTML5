HTMLの書き方の基本ルール・文書構造・DOCTYPE・文字参照・コメントを学びましょう。

① 基本的な書式と各部の名称

ポイント
HTMLは タグ を使ってコンテンツに意味を付けます。
タグの構成要素にはそれぞれ名称があり、試験でも問われます。

▼ 要素の基本構造（各部の名称）

開始タグ
<a href="https://example.com" target="_blank"> リンクテキスト </a> 終了タグ

↑要素名 ↑属性名 ↑属性値 ↑属性名 ↑属性値 ↑コンテンツ

名称説明例
要素（element）開始タグ〜終了タグ全体のこと<p>本文</p> 全体
開始タグ要素の始まりを示す<p>
終了タグ要素の終わりを示す（/が付く）</p>
要素名（タグ名）タグの種類を示す名前p, a, div
属性（attribute）要素に追加情報を付けるhref="..."
属性名属性の種類href, class, id
属性値属性の値（ダブルクォートで囲む）"https://example.com"
コンテンツ開始〜終了タグの間の内容リンクテキスト
空要素コンテンツを持たない要素（終了タグなし）<br>, <img>, <hr>

空要素の書き方

ポイント②：空要素は <○○> と書いても <○○/> と書いてもOK 
<!-- どちらも正しい書き方 -->
<br>          <!-- HTML5 推奨スタイル -->
<br/>         <!-- XHTMLスタイル（HTML5でも有効） -->
<img src="photo.jpg" alt="写真">   <!-- OK -->
<img src="photo.jpg" alt="写真"/>  <!-- OK -->
<hr>          <!-- OK -->
<hr/>         <!-- OK -->
⚠️ 試験重要ポイント：空白文字に注意！
HTML5以降、/ と > の間に空白文字を入れることはできません。

✅ <br/>　← 正しい（スラッシュと>の間に空白なし）
❌ <br />　← HTML5では不正（スラッシュと>の間に空白あり）

また、空要素には終了タグ（</br>など）を指定できません。

② HTMLの全体構造

ポイント
HTML文書は決まった構造を持っています。各要素の役割を理解しましょう。 
<!DOCTYPE html>                          <!-- DOCTYPE宣言 -->
<html lang="ja">                        <!-- ルート要素 -->

  <head>                                 <!-- メタ情報セクション -->
    <meta charset="UTF-8">             <!-- 文字コード指定 -->
    <meta name="viewport"
          content="width=device-width,
          initial-scale=1.0">            <!-- レスポンシブ対応 -->
    <title>ページタイトル</title>          <!-- タブに表示されるタイトル -->
    <link rel="stylesheet" href="style.css"> <!-- 外部CSS -->
  </head>

  <body>                                 <!-- 表示コンテンツセクション -->
    <h1>見出し</h1>
    <p>本文テキスト</p>
  </body>

</html>

要素役割
<!DOCTYPE html>HTML5文書であることをブラウザに宣言する
<html>HTML文書全体のルート（根）要素
<head>ブラウザ向けのメタ情報（画面に表示されない）
<meta charset>文字コードを指定（UTF-8が標準）
<title>ブラウザのタブ・検索結果に表示されるタイトル
<body>ブラウザに表示されるコンテンツ全体

タグの省略ルール

ポイント①：タグを省略できる要素がある
⚠️ 試験重要ポイント：省略できる要素は限られている！

開始タグ・終了タグの両方を省略できる要素：
html　head　body　colgroup　tbody　のみ

終了タグのみ省略できる要素（例）：
p　li　dt　dd　tr　th　td　など

省略できない要素：
title　script　div　span　など多数 
<!-- タグ省略の例：以下は有効なHTML5 -->

<!-- html / head / body タグを省略した書き方 -->
<!DOCTYPE html>
<title>省略サンプル</title>
<p>本文（bodyタグなし）

<!-- p要素は終了タグを省略できる -->
<p>段落1    <!-- </p> を省略 -->
<p>段落2    <!-- 次の <p> が来たら前の段落は自動的に閉じる -->
<p>段落3
✅ ただし、省略できても明示的に書くことが推奨されます。
可読性・保守性のために、実務では省略しないのが一般的です。

③ DOCTYPE宣言

ポイント③：DOCTYPE の基本形は <!DOCTYPE html> だが、これ以外も指定可能 
<!-- HTML5（現在の標準・推奨） -->
<!DOCTYPE html>

<!-- HTML 4.01 Strict（過去の書き方） -->
<!DOCTYPE HTML PUBLIC
  "-//W3C//DTD HTML 4.01//EN"
  "http://www.w3.org/TR/html4/strict.dtd">

<!-- HTML 4.01 Transitional（過去の書き方） -->
<!DOCTYPE HTML PUBLIC
  "-//W3C//DTD HTML 4.01 Transitional//EN"
  "http://www.w3.org/TR/html4/loose.dtd">

<!-- XHTML 1.0 Strict（過去の書き方） -->
<!DOCTYPE html PUBLIC
  "-//W3C//DTD XHTML 1.0 Strict//EN"
  "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
DOCTYPEバージョン特徴
<!DOCTYPE html> HTML5（現在） シンプル・短い・推奨
<!DOCTYPE HTML PUBLIC "..."> HTML 4.01 DTDのURLを指定する長い書き方
<!DOCTYPE html PUBLIC "..."> XHTML 1.0 XMLベースのHTML。厳密な構文が必要
⚠️ DOCTYPE宣言がない場合
ブラウザが「後方互換モード（Quirks Mode）」で動作し、
CSSのレイアウトが意図しない表示になることがあります。
必ず文書の1行目に記述してください。

④ 文字参照

ポイント
HTMLの特殊文字（< や & など）をそのまま書くとタグと混同されます。
文字参照を使って安全に表示しましょう。
書き方は 名前付き文字参照（&lt;）と 数値文字参照（&#60; / &#x3C;）の2種類があります。
表示したい文字 名前付き文字参照 10進数値参照 16進数値参照 用途
< &lt; &#60; &#x3C; タグの開始と混同を避ける
> &gt; &#62; &#x3E; タグの終了と混同を避ける
& &amp; &#38; &#x26; 文字参照の開始記号のエスケープ
" &quot; &#34; &#x22; 属性値内のダブルクォート
' &apos; &#39; &#x27; 属性値内のシングルクォート
 （スペース） &nbsp; &#160; &#xA0; 改行されないスペース（ノーブレークスペース）
© &copy; &#169; &#xA9; 著作権マーク 
<!-- 文字参照の使用例 -->
<p>HTMLでは &lt;p&gt; のようにタグを書きます。</p>
<!-- 表示結果：HTMLでは <p> のようにタグを書きます。 -->

<p>著作権 &copy; 2025 Example Corp.</p>
<!-- 表示結果：著作権 (c) 2025 Example Corp. -->

<p>価格：1&nbsp;000円（ノーブレークスペース）</p>
<!-- 「1」と「000円」の間で改行されない -->

▼ 実際の表示確認

タグの書き方：<p>本文</p>

数式：1 < 2 < 3 > 0

著作権：© 2025 Example Corp.

引用："Hello, World!"

⑤ コメント

ポイント
コメントはブラウザに表示されない注釈です。
<!-- で始まり --> で終わります。 
<!-- これはコメントです。ブラウザには表示されません -->

<!-- 複数行のコメントも書けます
     この行も
     この行もコメントです -->

<p>表示されるテキスト</p>
<!-- <p>コメントアウトされた段落（表示されない）</p> -->

<!-- ハイフン2つ「--」はコメント内に含めることができます -->
<!-- 例：2024-04-01 -- 修正履歴のメモ -->
⚠️ コメントの注意点（HTML5仕様）

コメントのテキスト部分には以下の制約があります。

① コメントテキストは次の文字列で開始できない：
　❌ > で始まる　（例：<!-- >コメント -->）
　❌ -> で始まる　（例：<!-- ->コメント -->）

② コメントテキストには次の文字列を含めることができない：
　❌ <!-- を含む　（コメントの開始記号と混同）
　❌ --> を含む　（コメントの終了記号と混同）
　❌ --! を含む

③ コメントテキストは次の文字列で終了できない：
　❌ <!- で終わる　（例：<!-- コメント<!- -->）

✅ --（ハイフン2つ）はそれ単体であれば使用できます。
✅ ブラウザの開発者ツールではコメントも見えます（機密情報を書かないこと）
✅ コメントの活用例
・セクションの区切りを明示する
・一時的にコードを無効化（コメントアウト）する
・TODO・修正履歴などのメモを残す

📌 重要ポイント まとめ

項目ルール・ポイント
空要素の書き方 <br> でも <br/> でもOK。ただし <br />（空白あり）は不正
空要素の終了タグ 空要素に終了タグ（</br>など）は指定できない

終了タグのみ省略可能 p / li / dt / dd / tr / th / td など

<!- で終了できない。
