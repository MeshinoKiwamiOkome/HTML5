① CSSの指定元による優先順位

ポイント
CSSは3つの「指定元（オリジン）」から適用されます。
優先度は 製作者 > ユーザー > ユーザーエージェント の順です。

3つの指定元（オリジン）

指定元説明例
ユーザーエージェント
（ブラウザ）		 ブラウザが持つデフォルトスタイル h1が大きく表示される・aが青色・ulに●マーク
ユーザー ブラウザの設定やユーザーが適用するスタイル ブラウザのフォントサイズ設定・ユーザースタイルシート
製作者 Webサイト制作者が書いたCSS link要素・style要素・style属性で指定したCSS

通常時の優先順位（高い順）

最高 🥇 製作者のCSS ← 最優先
中 🥈 ユーザーのCSS
最低 🥉 ユーザーエージェント（ブラウザ）のCSS ← 最低
⚠️ 試験重要ポイント①：!importantで優先順位が逆転する

通常は 製作者 > ユーザー > ユーザーエージェント の順ですが、
!important を付けると逆転します。

!important 付きの優先順位（高い順）：
ユーザーエージェント !important > ユーザー !important > 製作者 !important

!important 付きの優先順位（高い順）

最高 🔴 ユーザーエージェントの !important ← 逆転！
↑ 🟠 ユーザーの !important
↑ 🟡 製作者の !important
↓ 🟢 製作者の通常CSS
↓ 🔵 ユーザーの通常CSS
最低 🟣 ユーザーエージェントの通常CSS ← 最低
✅ 覚え方のポイント
・通常時：製作者が最優先（Webサイトのデザインを優先）
・!important 付き：逆転して ユーザーエージェント > ユーザー > 製作者 の順になる
・ユーザーの !important が製作者より優先されるのは、 アクセシビリティ（視覚障害者の大文字設定など）のため

② !importantで優先順位を高くする

ポイント
・!important は宣言の値の後ろに記述します。
・同じ指定元内では !important が付いた宣言が最優先になります。
・!important 同士が競合した場合は詳細度で決まります。 
/* !important の書き方 */
p {
    color: red !important;   /* !important を付ける */
    font-size: 16px;           /* こちらは通常の宣言 */
}

/* 詳細度が低くても !important で勝てる例 */
p       { color: blue; }           /* 詳細度 0-0-1 */
#main p { color: green; }          /* 詳細度 1-0-1（高い） */
p       { color: red !important; } /* !important → 赤が適用される */

/* !important 同士が競合した場合は詳細度で決まる */
p       { color: red   !important; } /* 詳細度 0-0-1 */
.text p { color: blue  !important; } /* 詳細度 0-1-1 → 青が適用 */
⚠️ !importantの注意点
・!important の多用はCSSの管理を複雑にします
・!important を上書きするには、さらに !important が必要になり 「!important地獄」になりがちです
・使用は最小限にとどめ、詳細度の設計で解決するのが推奨です
・アクセシビリティ対応（ユーザースタイルシート）や サードパーティCSSの上書きなど、やむを得ない場合に限定して使います

同じ指定元・同じ詳細度の場合は後から書いた方が優先

/* 同じ詳細度（0-0-1）の場合、後に書いた方が適用される */
p { color: red; }    /* ← こちらは上書きされる */
p { color: blue; }   /* ← 後に書いた方が適用される */

/* 外部CSSとstyle要素の順序も同様 */
<link rel="stylesheet" href="base.css">   <!-- 先に読み込まれる -->
<link rel="stylesheet" href="theme.css">  <!-- 後に読み込まれる → 優先 -->

③ セレクタの詳細度による優先順位の計算方法

ポイント②③：詳細度の仕組み
・詳細度が最も高いのは style属性による指定（インラインスタイル）
・セレクタの詳細度は IDセレクタ（A）・属性系セレクタ（B）・要素系セレクタ（C）の3桁で示します

詳細度の4段階（高い順）

最高 style属性（インラインスタイル） 1-0-0-0

中 クラス・属性・擬似クラスセレクタ 0-0-1-0

🧮 詳細度 インタラクティブ計算機

セレクタを入力すると詳細度を計算します

← セレクタを入力してください

詳細度の比較ルール

/* 詳細度の比較は左の桁（ID）から順に比較する */

/* 例1：IDセレクタ vs クラスセレクタ多数 */
#id                        { color: red; }   /* 1-0-0 */
.a.b.c.d.e.f.g.h.i.j.k   { color: blue; }  /* 0-11-0 → 赤が適用！ */
/* → IDの桁（1 vs 0）で比較するため、クラスが何個あっても#idが勝つ */

/* 例2：詳細度が同じ場合は後に書いた方が適用 */
.btn.primary  { color: blue; }   /* 0-2-0 */
.nav .item    { color: red; }    /* 0-2-0 → 後に書いた赤が適用 */
セレクタの種類詳細度への影響例
style属性1-0-0-0（最高・別格）style="color:red"
IDセレクタA桁に +1#header
クラスセレクタB桁に +1.nav
属性セレクタB桁に +1[type="text"]
擬似クラスB桁に +1:hover / :nth-child()
要素セレクタ（タイプ）C桁に +1p / h1
擬似要素C桁に +1::before / ::first-line
ユニバーサルセレクタ影響なし（0-0-0）*
結合子影響なしスペース / > / + / ~
:not() 自体影響なし（中身は計算）:not(.hidden)
:is() / :has()引数の中で最も高い詳細度:is(#id, .cls) → 1-0-0
:where()常に 0-0-0:where(h1, h2)

④ 優先順位の総まとめ（高い順）

①最高 ユーザーエージェントの !important 逆転
② ユーザーの !important 逆転
③ 製作者の !important ↑ !important
④ 製作者：style属性（インライン） 詳細度最高
⑤ 製作者：IDセレクタ（#id） A桁
⑥ 製作者：クラス・属性・擬似クラス B桁
⑦ 製作者：要素・擬似要素 C桁
⑧ ユーザーの通常CSS
⑨最低 ユーザーエージェントの通常CSS 最低

※ 同じ優先度レベル内では詳細度 → 記述順（後が優先）で決まります

Q5. コード穴埋め問題

以下のコードの __A__ 〜 __E__ に 適切な詳細度・値・説明を入力してください。

/* A：このセレクタの詳細度は？ */

/* 詳細度 → __A__ */

/* B：このセレクタの詳細度は？ */
.card.active::before { content: "★"; }
/* 詳細度 → __B__ */

/* C：!importantで優先順位を上げる書き方 */

📌 CSS適用の優先順位 まとめ

優先順位内容重要ポイント
指定元（通常） 製作者 > ユーザー > UA 製作者のCSSが最優先
指定元（!important） UA !important > ユーザー !important > 製作者 !important !importantで逆転。アクセシビリティのため
style属性 詳細度 1-0-0-0（別格） 製作者CSSの中で最高詳細度
詳細度 A桁 IDセレクタ（#id） 1IDにつき +1
詳細度 B桁 クラス・属性セレクタ・擬似クラス 1つにつき +1。:not()自体は0
詳細度 C桁 要素セレクタ・擬似要素 1つにつき +1
詳細度なし ユニバーサルセレクタ・結合子 詳細度に影響しない（0-0-0）
同詳細度の場合 後に記述したスタイルが優先 カスケード（上書き）の基本
