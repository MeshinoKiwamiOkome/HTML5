② p要素 / a要素

<!-- p：段落を表すブロック要素 -->
<p>これは段落です。段落はブロックレベル要素で、前後に改行が入ります。</p>
<p>2つ目の段落です。</p>

<!-- a：ハイパーリンク -->
<a href="https://example.com">外部リンク</a>
<a href="page.html">内部リンク</a>
<a href="#section">ページ内リンク</a>
<a href="mailto:info@example.com">メールリンク</a>
<a href="file.pdf" download>ダウンロードリンク</a>
<a href="https://example.com" target="_blank"
   rel="noopener noreferrer">新しいタブで開く</a>
⚠️ target="_blank" のセキュリティ対策
target="_blank" を使う場合は rel="noopener noreferrer" を必ず指定しましょう。
指定しないと開いたページから元のページを操作できる脆弱性（タブナビング）が生じます。

③ em要素 / strong要素

意味的な強調と重要性の違い
・em：テキストに強調（アクセント）を与える。読み上げ時に強調される。
・strong：テキストの重要性・深刻さ・緊急性を示す。 
<!-- em：強調（斜体で表示される） -->
<p>この操作は<em>元に戻せません</em>。</p>

<!-- strong：重要性（太字で表示される） -->
<p><strong>警告：</strong>このファイルを削除すると復元できません。</p>

<!-- 組み合わせも可能 -->
<p><strong><em>絶対に</em></strong>触れないでください。</p>

この操作は元に戻せません。

警告：このファイルを削除すると復元できません。

絶対に触れないでください。

⚠️ b / i との違い
・em は意味的な強調 → i は視覚的な斜体（意味なし）
・strong は意味的な重要性 → b は視覚的な太字（意味なし）

④ blockquote要素 / q要素 / cite要素

<!-- blockquote：ブロックレベルの引用 -->
<blockquote cite="https://example.com/article">
  <p>HTMLはウェブページの構造を定義するマークアップ言語です。</p>
</blockquote>

<!-- q：インラインの引用（引用符が自動で付く） -->
<p>彼は<q cite="https://example.com">HTMLを学ぼう</q>と言った。</p>

<!-- cite：作品・文書のタイトル（斜体で表示） -->
<p><cite>HTML5プロフェッショナル認定試験 公式テキスト</cite>を参照。</p>

HTMLはウェブページの構造を定義するマークアップ言語です。

彼はHTMLを学ぼうと言った。

HTML5プロフェッショナル認定試験 公式テキストを参照。

⚠️ cite属性 と cite要素 の違い
・cite属性：blockquote / q の引用元URLを指定する属性
・<cite>要素：書籍・映画・記事などの作品タイトルを表す要素（人名には使わない）

⑤ mark要素 / small要素

ポイント①：mark要素は「注目してもらうために目立たせている部分」を表す
・検索結果のキーワードハイライト・重要箇所のマーカーなどに使います。
・単なる装飾目的のハイライトには使いません（CSSを使うべき）。 
<!-- mark：注目・ハイライト（黄色背景で表示） -->
<p>検索結果：<mark>HTML5</mark>の学習方法について</p>
<p>試験では<mark>セクション要素は4つ</mark>という点が頻出です。</p>

<!-- small：注釈・免責事項・著作権表示など -->
<p>月額980円<small>（税込・初月無料）</small></p>
<footer>
  <small>&copy; 2025 MeshinoKiwamiOkome</small>
</footer>

検索結果：HTML5の学習方法について

試験ではセクション要素は4つという点が頻出です。

月額980円（税込・初月無料）

⑥ data要素 / time要素

ポイント②③：data要素とtime要素
・data：機械読み取りが可能なデータを value 属性として提供する要素
・time：data 要素を日時に特化させた要素。datetime 属性に機械読み取り用の日時を指定 
<!-- data：機械読み取り用データをvalue属性に指定 -->
<p>商品コード：
  <data value="P-12345">ノートパソコン</data>
</p>
<p>価格：
  <data value="89800">89,800円</data>
</p>

<!-- time：日時に特化したdata要素 -->
<p>公開日：
  <time datetime="2025-03-26">2025年3月26日</time>
</p>
<p>開始時刻：
  <time datetime="2025-03-26T14:00">午後2時</time>
</p>
<p>所要時間：
  <time datetime="PT2H30M">2時間30分</time>
</p>

商品コード：ノートパソコン

公開日：2025年3月26日

開始時刻：午後2時

所要時間：2時間30分

比較data要素time要素
用途任意の機械読み取り用データ日時・期間に特化
属性value（必須）datetime（任意）
値の形式任意の文字列ISO 8601形式の日時

⑦ abbr要素 / dfn要素

<!-- abbr：略語・頭字語（title属性で正式名称を指定） -->
<p>
  <abbr title="HyperText Markup Language">HTML</abbr>は
  <abbr title="World Wide Web Consortium">W3C</abbr>が策定しました。
</p>

<!-- dfn：用語の定義（その文書内での初出定義に使う） -->
<p>
  <dfn>セマンティクス</dfn>とは、
  HTMLタグに意味を持たせることで文書構造を明確にする考え方です。
</p>

<!-- dfn + abbr の組み合わせ -->
<p>
  <dfn><abbr title="Cascading Style Sheets">CSS</abbr></dfn>は
  スタイルを定義する言語です。
</p>

HTMLは W3Cが策定しました。

セマンティクスとは、HTMLタグに意味を持たせることで文書構造を明確にする考え方です。

⑧ b要素 / i要素 / s要素 / u要素

意味的な要素との使い分けが重要
これらは視覚的な表現に加え、それぞれ固有の意味を持ちます。 単なる装飾目的にはCSSを使いましょう。 
<!-- b：注目させたい部分（重要性・強調の意味なし） -->
<p>手順：<b>「設定」</b>を開き、<b>「プライバシー」</b>を選択。</p>

<!-- i：専門用語・慣用句・外国語・技術用語など -->
<p>ラテン語で<i lang="la">et cetera</i>（など）と表記します。</p>

<!-- s：もはや正確でない・関連性のない内容（取り消し線） -->
<p>通常価格：<s>10,000円</s> → セール価格：7,800円</p>

<!-- u：注釈的な下線（スペルミスの指摘など） -->
<p>この単語の<u>スペルに注意</u>してください。</p>

手順：「設定」を開き、「プライバシー」を選択。

ラテン語でet cetera（など）と表記します。

通常価格：10,000円 → セール価格：7,800円

この単語のスペルに注意してください。

要素意味混同しやすい要素
b注目させたい部分（重要性・強調なし）strong（重要性あり）
i専門語・慣用句・外国語・技術用語em（強調あり）
sもはや正確でない・関連性のない内容del（編集上の削除）
u注釈的な下線・スペルミスの指摘CSSのtext-decoration

⑨ bdo要素 / bdi要素

<!-- bdo：書字方向を強制的に上書き -->
<p><bdo dir="rtl">Hello, World!</bdo></p>
<!-- 表示：!dlroW ,olleH -->

<!-- bdi：周囲のテキストから書字方向を分離 -->
<!-- ユーザー名など、書字方向が不明なテキストを含む場合に使用 -->
<ul>
  <li>1位：<bdi>山田太郎</bdi> - 100点</li>
  <li>2位：<bdi>Ahmed Ali</bdi> - 95点</li>
</ul>

bdo（rtl）：Hello, World!

  • 1位：山田太郎 - 100点
  • 2位：Ahmed Ali - 95点
✅ bdo と bdi の使い分け
・bdo：書字方向を強制的に指定したいとき（dir 属性必須）
・bdi：書字方向が不明なテキスト（ユーザー名など）を周囲から分離したいとき

⑩ pre / code / kbd / samp / var要素

<!-- pre：整形済みテキスト（空白・改行をそのまま表示） -->
<pre>
  function hello() {
    console.log("Hello!");
  }
</pre>

<!-- code：コードの断片（インライン） -->
<p><code>console.log()</code>でコンソールに出力できます。</p>

<!-- pre + code：コードブロック（組み合わせが一般的） -->
<pre><code>
const x = 10;
const y = 20;
console.log(x + y); // 30
</code></pre>

<!-- kbd：キーボード入力 -->
<p>保存するには<kbd>Ctrl</kbd> + <kbd>S</kbd>を押してください。</p>

<!-- samp：プログラムの出力結果 -->
<p>実行結果：<samp>Hello, World!</samp></p>

<!-- var：変数・引数・数式の変数 -->
<p>面積 <var>S</var> = <var>a</var> × <var>b</var></p>

console.log()でコンソールに出力できます。

保存するにはCtrl + Sを押してください。

実行結果：Hello, World!

面積 S = a × b

⑪ sup要素 / br要素 / wbr要素

<!-- sup：上付き文字 -->
<p>E = mc<sup>2</sup></p>
<p>注釈<sup>*1</sup>を参照してください。</p>

<!-- br：改行（空要素）-->
<p>〒700-0000<br>△△県■■市○○1-2-3</p>

<!-- wbr：改行可能位置のヒント（空要素）-->
<!-- 長いURLや単語の折り返し位置を指定 -->
<p>https://www.example.com<wbr>/very<wbr>/long<wbr>/path</p>

E = mc2

注釈*1を参照してください。

〒000-0000
△△県■■市○○1-2-3

⚠️ br要素の注意点
・br は段落の区切りには使いません（p 要素を使う）
・住所・詩・歌詞など、改行自体がコンテンツの一部である場合に使います
・レイアウト目的の改行はCSSで対応します

⑫ ins要素 / del要素

<!-- ins：文書に追加されたテキスト（下線で表示） -->
<!-- del：文書から削除されたテキスト（取り消し線で表示） -->
<p>
  開催日：
  <del datetime="2025-03-20">3月20日</del>
  <ins datetime="2025-03-26">3月26日</ins>
</p>

<!-- cite属性で変更理由のURLを指定できる -->
<del cite="https://example.com/changelog"
     datetime="2025-01-01">
  旧仕様のテキスト
</del>

開催日： 3月20日 3月26日

⚠️ s要素 と del要素 の違い
・s：もはや正確でない・関連性のない内容（セール価格の旧価格など）
・del：文書の編集履歴として削除されたテキスト（変更履歴の管理）
　datetime 属性で削除日時、cite 属性で変更理由のURLを指定できます
