① セレクタの種類と組み合わせのルール

/* ===== セレクタの種類一覧 ===== */

p           { } /* タイプセレクタ：要素名で指定 */
*           { } /* ユニバーサルセレクタ：すべての要素 */
.class      { } /* クラスセレクタ */
#id         { } /* IDセレクタ */
[attr]      { } /* 属性セレクタ */
:hover      { } /* 擬似クラス */
::before    { } /* 擬似要素 */

/* ===== 組み合わせのルール ===== */

/* グループ化（カンマ区切り）：複数セレクタに同じスタイル */
h1, h2, h3 { color: #553c9a; }

/* 複合セレクタ（スペースなし連結）：条件を重ねる */
p.highlight        { } /* p要素 かつ .highlight クラス */
input[type="text"] { } /* input要素 かつ type="text" */
a:hover            { } /* a要素 かつ :hover 状態 */

② タイプセレクタ / ユニバーサルセレクタ

④ 属性セレクタ

ポイント①：属性セレクタの記号はしっかりと覚える
記号の違いで「完全一致」「前方一致」「後方一致」「部分一致」などが変わります。
[attr]
属性が存在する要素
例：[disabled]
[attr="val"]
属性値が完全一致
例：[type="text"]
[attr~="val"]
スペース区切りの値の1つに一致
例：[class~="btn"]
[attr|="val"]
値が完全一致 または 「val-」で始まる
例：[lang|="en"] → en, en-US
[attr^="val"]
値が「val」で始まる（前方一致）
例：[href^="https"]
[attr$="val"]
値が「val」で終わる（後方一致）
例：[href$=".pdf"]
[attr*="val"]
値に「val」が含まれる（部分一致）
例：[href*="example"]
[attr="val" i]
大文字・小文字を区別しない
例：[type="TEXT" i]
/* 属性セレクタの使用例 */

/* [attr]：属性が存在する */
input[disabled]     { opacity: 0.5; }
a[target]           { color: orange; }

/* [attr="val"]：完全一致 */
input[type="text"]  { border: 2px solid #9f7aea; }
input[type="email"] { border: 2px solid #48bb78; }

/* [attr^="val"]：前方一致（^はキャレット） */
a[href^="https"]    { color: green; }    /* httpsリンク */
a[href^="mailto"]   { color: blue; }     /* メールリンク */
a[href^="tel"]      { color: purple; }   /* 電話リンク */

/* [attr$="val"]：後方一致（$はドル記号） */
a[href$=".pdf"]     { color: red; }      /* PDFリンク */
a[href$=".zip"]     { color: orange; }   /* ZIPリンク */

/* [attr*="val"]：部分一致（*はアスタリスク） */
a[href*="example"]  { font-weight: bold; }

/* [attr~="val"]：スペース区切りの値の1つに一致（~はチルダ） */
[class~="btn"]      { cursor: pointer; }

/* [attr|="val"]：完全一致またはval-で始まる（|はパイプ） */
[lang|="en"]        { font-family: serif; } /* en, en-US, en-GB */

▼ 属性セレクタのライブデモ

httpsリンク（緑色・太字）　 httpリンク（通常）　 PDFリンク（📄アイコン付き）

⚠️ 記号の覚え方
・^（キャレット）= 前方一致　← 「^」は文字列の先頭を表す正規表現と同じ
・$（ドル）= 後方一致　← 「$」は文字列の末尾を表す正規表現と同じ
・*（アスタリスク）= 部分一致　← 「何でも」を表すワイルドカードと同じ
・~（チルダ）= スペース区切りの値の1つ
・|（パイプ）= 完全一致またはハイフン区切りの先頭

⑤ 擬似クラス

ポイント②：擬似クラスの式「an+b」の指定方法をしっかりと覚える
:nth-child(an+b) の a・n・b の意味を理解しましょう。

リンク関連の擬似クラス

/* リンクの状態を表す擬似クラス（LVHA の順で記述するのが推奨） */
a:link    { color: blue; }      /* 未訪問リンク */
a:visited { color: purple; }    /* 訪問済みリンク */
a:hover   { color: red; }       /* マウスオーバー */
a:active  { color: orange; }    /* クリック中 */
a:focus   { outline: 2px solid blue; } /* フォーカス時 */

構造的擬似クラス

/* 位置・順序による選択 */
li:first-child       { } /* 最初の子要素 */
li:last-child        { } /* 最後の子要素 */
li:only-child        { } /* 唯一の子要素 */
p:first-of-type     { } /* 同種の最初の要素 */
p:last-of-type      { } /* 同種の最後の要素 */
p:only-of-type      { } /* 同種の唯一の要素 */
:root               { } /* ルート要素（html要素） */
p:empty             { } /* 子要素・テキストがない要素 */

an+b 式（nth系擬似クラス）

/* :nth-child(an+b) の式の意味 */
/* a = 間隔（何個おき）, n = 0,1,2,3... , b = オフセット（開始位置） */

li:nth-child(2n)    { } /* 偶数番目（2,4,6...）even と同じ */
li:nth-child(2n+1)  { } /* 奇数番目（1,3,5...）odd と同じ */
li:nth-child(3n)    { } /* 3の倍数番目（3,6,9...） */
li:nth-child(3n+1)  { } /* 1,4,7,10... */
li:nth-child(n+3)   { } /* 3番目以降すべて */
li:nth-child(-n+3)  { } /* 最初の3つ（1,2,3） */
li:nth-child(1)     { } /* 1番目のみ（:first-child と同じ） */
li:nth-child(even)  { } /* 偶数番目（even キーワード） */
li:nth-child(odd)   { } /* 奇数番目（odd キーワード） */

/* :nth-last-child：末尾から数える */
li:nth-last-child(2)  { } /* 末尾から2番目 */

/* :nth-of-type：同種の要素の中でのn番目 */
p:nth-of-type(2n)    { } /* p要素の中で偶数番目 */

▼ nth-child のライブデモ

  • 1番目（:first-child → 紫・太字）
  • 2番目（:nth-child(2n) → 背景薄紫）
  • 3番目（:nth-child(3n+1) → 左ボーダー）
  • 4番目（:nth-child(2n) → 背景薄紫）
  • 5番目（:last-child → 赤・太字）
式選択される要素キーワード
2n2, 4, 6, 8 …（偶数）even
2n+11, 3, 5, 7 …（奇数）odd
3n3, 6, 9, 12 …—
3n+11, 4, 7, 10 …—
n+33番目以降すべて—
-n+31, 2, 3（最初の3つ）—
11番目のみ—

フォーム関連の擬似クラス

input:enabled   { } /* 有効な入力要素 */
input:disabled  { } /* 無効な入力要素 */
input:checked   { } /* チェックされたcheckbox/radio */
input:required  { } /* required属性がある入力要素 */

⑥ 擬似要素

ポイント③：擬似要素の先頭のコロンは、CSSから2つになった
・CSS2では :before（コロン1つ）でしたが、 CSS3以降では ::before（コロン2つ）が正式な書き方です。
・擬似クラス（:hover など）との区別のためにコロンが2つになりました。
・後方互換性のため :before（1つ）も多くのブラウザで動作しますが、 現在は ::before（2つ）が推奨です。 
/* ===== CSS2（旧） vs CSS3以降（現在） ===== */
p:before  { } /* CSS2の書き方（非推奨） */
p::before { } /* CSS3以降の正式な書き方（推奨） */

/* ===== 主な擬似要素 ===== */

/* ::before / ::after：要素の前後にコンテンツを挿入 */
.note::before {
    content: "📌 ";    /* contentプロパティは必須 */
}
.external::after {
    content: " ↗";
}
/* content: "" で空文字（装飾目的） */
.divider::after {
    content: "";
    display: block;
    border-bottom: 1px solid #ccc;
}

/* ::first-line：最初の行 */
p::first-line {
    font-weight: bold;
    color: #553c9a;
}

/* ::first-letter：最初の文字（ドロップキャップ） */
p::first-letter {
    font-size: 2em;
    float: left;
    color: #9f7aea;
}

/* ::selection：テキスト選択時のスタイル */
::selection {
    background: #9f7aea;
    color: #fff;
}

/* ::placeholder：プレースホルダーのスタイル */
input::placeholder {
    color: #a0aec0;
    font-style: italic;
}

/* ::marker：リストマーカーのスタイル */
li::marker {
    color: #9f7aea;
    font-size: 1.2em;
}

▼ 擬似要素のライブデモ

これは ::first-line（最初の行が紫・太字）と ::first-letter（最初の文字が大きく紫）のデモです。テキストが複数行になると効果がよくわかります。

before・afterで括弧を追加

擬似要素説明CSS2記法（非推奨）
::before要素の直前にコンテンツ挿入:before
::after要素の直後にコンテンツ挿入:after
::first-line最初の行:first-line
::first-letter最初の文字:first-letter
::selection選択されたテキストなし（CSS3追加）
::placeholderプレースホルダーなし（CSS3追加）
::markerリストマーカーなし（CSS4追加）

⑦ 結合子

ポイント①：結合子の記号はしっかりと覚える
結合子はセレクタとセレクタの「関係性」を指定します。
A B
子孫結合子
（スペース）
AのすべてのB子孫
A > B
子結合子
（>）
Aの直接の子Bのみ
A + B
隣接兄弟結合子
（+）
Aの直後の兄弟B
A ~ B
一般兄弟結合子
（~）
Aの後ろにある兄弟Bすべて
/* 子孫結合子（スペース）：Aの中にあるすべてのB */
div p        { color: blue; }   /* div内のすべてのp（深さ問わず） */
.nav a       { color: white; }  /* .nav内のすべてのa */

/* 子結合子（>）：Aの直接の子Bのみ */
ul > li      { color: red; }    /* ulの直接の子liのみ（孫liは対象外） */
.parent > p { color: purple; }

/* 隣接兄弟結合子（+）：Aの直後の兄弟Bのみ */
h1 + p       { font-size: 1.2em; } /* h1の直後のpのみ */
label + input{ margin-left: 8px; }

/* 一般兄弟結合子（~）：Aの後ろにある兄弟Bすべて */
h1 ~ p       { color: gray; }    /* h1の後ろにあるすべてのp */
.active ~ li{ opacity: 0.5; }

▼ 結合子のライブデモ

子結合子（>）：.parent の直接の子p → 紫・太字

孫のp（子結合子の対象外）

基準要素（.sibling）

隣接兄弟（+）：直後のp → 緑・斜体

基準要素（.sibling-all）

一般兄弟（~）：後ろのpすべて → 下線

一般兄弟（~）：後ろのpすべて → 下線

結合子記号選択対象例
子孫結合子スペースAの中にあるすべてのB（深さ問わず）div p
子結合子>Aの直接の子Bのみul > li
隣接兄弟結合子+Aの直後の兄弟Bのみ（1つ）h1 + p
一般兄弟結合子~Aの後ろにある兄弟Bすべてh1 ~ p
