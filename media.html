🖼️ HTML5 画像・動画・音声

img / picture / source / video / audio / track / embed / map / area / object の役割と使い方を学びましょう。

① img要素

ポイント
・<img> は画像を埋め込む空要素です。
・src 属性（画像URL）と alt 属性（代替テキスト）が基本セットです。
・width / height 属性でサイズを指定できますが、 パーセント値は指定できません（ピクセル値のみ）。
⚠️ 試験重要ポイント①：alt属性の省略

通常、alt 属性は必須ですが、以下の特別なケースでは省略できます。

✅ 省略できるケース：
　・画像がキャプション（<figcaption> など）によって説明されている場合
　・純粋に装飾目的の画像で、コンテンツ上の意味を持たない場合 　（この場合は alt="" と空文字を指定するのが推奨）

❌ 省略してはいけないケース：
　・画像がコンテンツとして意味を持つ場合
　・画像がリンクになっている場合（リンク先の説明をaltに書く）

⚠️ alt 属性を省略する（属性自体を書かない）のと、 空文字を指定する（alt=""）は意味が異なります。
　・省略：スクリーンリーダーがファイル名を読み上げる場合がある
　・空文字：スクリーンリーダーがその画像をスキップする
⚠️ 試験重要ポイント③：width / height 属性の値

width 属性と height 属性に指定できる値は ピクセル値（整数）のみです。

✅ <img width="300" height="200">　← 正しい（単位なし整数）
❌ <img width="300px" height="200px">　← 不正（px単位は書かない）
❌ <img width="50%" height="50%">　← 不正（パーセント値は指定不可）

レスポンシブなサイズ調整は CSS の width / max-width で行います。 
<!-- 基本的な書き方 -->
<img src="photo.jpg"
     alt="富士山の写真"
     width="800"
     height="600">

<!-- 装飾目的の画像（altは空文字） -->
<img src="decoration.png" alt="">

<!-- figcaptionで説明されている場合はaltを省略可 -->
<figure>
  <img src="chart.png">
  <figcaption>2025年度 売上グラフ</figcaption>
</figure>

<!-- loading属性：遅延読み込み（パフォーマンス最適化） -->
<img src="photo.jpg" alt="写真" loading="lazy">
属性説明備考
src画像ファイルのURL必須
alt代替テキスト特別なケースを除き必須
width表示幅（ピクセル整数値のみ）パーセント値不可
height表示高さ（ピクセル整数値のみ）パーセント値不可
srcset解像度・幅に応じた複数画像候補pictureと組み合わせ可
sizes表示サイズのヒントsrcsetと併用
loading読み込みタイミング（lazy / eager）遅延読み込み対応
decodingデコードタイミング（async / sync / auto）パフォーマンス調整

② picture要素 / source要素（画像）

ポイント②：複数の候補画像の中から最適なものを表示させる機能
・<picture> は複数の画像候補を定義し、 ブラウザが最適なものを選んで表示する要素です。
・画面サイズ・解像度・対応フォーマットに応じた画像の出し分けができます。
・必ず最後に <img> 要素（フォールバック）を含める必要があります。
⚠️ 試験重要ポイント：picture内のsource要素の必須属性

<picture> 内の <source> 要素には srcset 属性が必須です。
また、media 属性または type 属性の いずれかが指定されていれば問題ありません。

✅ srcset + media 属性の組み合わせ
✅ srcset + type 属性の組み合わせ
✅ srcset のみでも動作する（media/typeは任意）

※ video / audio 内の source 要素とは 必須属性が異なる点に注意！ 
<!-- メディアクエリによる画像の出し分け -->
<picture>
  <!-- 幅768px以上ではPC用画像 -->
  <source media="(min-width: 768px)"
          srcset="photo-large.jpg">
  <!-- 幅480px以上ではタブレット用画像 -->
  <source media="(min-width: 480px)"
          srcset="photo-medium.jpg">
  <!-- フォールバック（必須） -->
  <img src="photo-small.jpg" alt="風景写真">
</picture>

<!-- フォーマットによる出し分け（type属性） -->
<picture>
  <!-- WebPに対応していればWebPを使用 -->
  <source type="image/webp"
          srcset="photo.webp">
  <!-- AVIFに対応していればAVIFを使用 -->
  <source type="image/avif"
          srcset="photo.avif">
  <!-- フォールバック -->
  <img src="photo.jpg" alt="風景写真">
</picture>

<!-- srcsetによる解像度対応（img単体でも可） -->
<img src="photo.jpg"
     srcset="photo.jpg 1x, photo@2x.jpg 2x, photo@3x.jpg 3x"
     alt="風景写真">

📌 picture内のsource

必須属性：srcset
任意属性：media または type
（どちらか一方でもOK・両方でもOK・なくてもOK）

📌 video/audio内のsource

必須属性：src
任意属性：type
（srcsetは使わない）

③ video要素

ポイント
・<video> は動画を埋め込む要素です。
・src 属性で直接指定するか、子要素の <source> で複数フォーマットを指定します。
・ブラウザが対応していない場合のフォールバックテキストを内部に書けます。 
<!-- 基本的な書き方 -->
<video src="movie.mp4"
       controls
       width="640"
       height="360">
  お使いのブラウザは動画再生に対応していません。
</video>

<!-- 複数フォーマットの指定（source要素） -->
<video controls width="640" height="360"
       poster="thumbnail.jpg">
  <!-- video内のsource要素はsrc属性が必須 -->
  <source src="movie.mp4"  type="video/mp4">
  <source src="movie.webm" type="video/webm">
  <source src="movie.ogv"  type="video/ogg">
  お使いのブラウザは動画再生に対応していません。
</video>
属性説明
src動画ファイルのURL（source要素を使う場合は不要）
controls再生コントロールを表示（論理属性）
autoplay自動再生（論理属性・muted必須のブラウザが多い）
mutedミュート状態で再生（論理属性）
loopループ再生（論理属性）
poster再生前に表示するサムネイル画像のURL
preload読み込み方法（none / metadata / auto）
width / height表示サイズ（ピクセル整数値のみ）
playsinlineiOSでインライン再生（論理属性）

④ audio要素

ポイント
・<audio> は音声を埋め込む要素です。
・基本的な構造は video 要素と同じです。
・width / height 属性はありません（音声のみのため）。 
<!-- 基本的な書き方 -->
<audio src="sound.mp3" controls>
  お使いのブラウザは音声再生に対応していません。
</audio>

<!-- 複数フォーマットの指定（source要素） -->
<audio controls>
  <!-- audio内のsource要素もsrc属性が必須 -->
  <source src="sound.mp3"  type="audio/mpeg">
  <source src="sound.ogg"  type="audio/ogg">
  <source src="sound.wav"  type="audio/wav">
  お使いのブラウザは音声再生に対応していません。
</audio>
属性説明
src音声ファイルのURL
controls再生コントロールを表示（論理属性）
autoplay自動再生（論理属性）
mutedミュート状態（論理属性）
loopループ再生（論理属性）
preload読み込み方法（none / metadata / auto）

⑤ track要素

ポイント
・<track> は video / audio 要素に 字幕・キャプション・チャプターなどのテキストトラックを追加する空要素です。
・WebVTT形式（.vtt）のファイルを指定します。
・kind 属性でトラックの種類を指定します。 
<video controls src="movie.mp4">
  <!-- 日本語字幕（デフォルト表示） -->
  <track kind="subtitles"
         src="subtitles-ja.vtt"
         srclang="ja"
         label="日本語"
         default>
  <!-- 英語字幕 -->
  <track kind="subtitles"
         src="subtitles-en.vtt"
         srclang="en"
         label="English">
  <!-- チャプター情報 -->
  <track kind="chapters"
         src="chapters.vtt"
         srclang="ja">
</video>
kind属性の値説明
subtitles字幕（翻訳テキスト）
captionsキャプション（音声説明・効果音なども含む）
descriptions映像の音声説明（視覚障害者向け）
chaptersチャプター情報（ナビゲーション用）
metadataスクリプトから利用するメタデータ

⑥ embed要素

ポイント
・<embed> はブラウザプラグインなどの外部コンテンツを埋め込む空要素です。
・PDFや外部アプリケーションのコンテンツを表示するために使います。
・現在は object 要素や iframe 要素が代替として使われることが多いです。 
<!-- PDFの埋め込み -->
<embed src="document.pdf"
       type="application/pdf"
       width="800"
       height="600">

<!-- 外部コンテンツの埋め込み -->
<embed src="https://example.com/plugin"
       type="application/x-shockwave-flash"
       width="400"
       height="300">
属性説明
src埋め込むコンテンツのURL
typeMIMEタイプ
width / height表示サイズ（ピクセル整数値のみ）

⑦ map要素 / area要素

ポイント
・<map> と <area> はイメージマップ（画像の特定領域をリンクにする）を定義します。
・<img> の usemap 属性で <map> の name 属性を参照します。
・<area> は空要素で、クリック可能な領域の形状・座標・リンク先を指定します。 
<!-- イメージマップの例 -->
<img src="japan-map.png"
     alt="日本地図"
     usemap="#japanmap"
     width="600"
     height="400">

<map name="japanmap">
  <!-- 矩形領域（左上x,y・右下x,y） -->
  <area shape="rect"
        coords="100,50,200,150"
        href="tokyo.html"
        alt="東京">
  <!-- 円形領域（中心x,y・半径） -->
  <area shape="circle"
        coords="300,200,50"
        href="osaka.html"
        alt="大阪">
  <!-- 多角形領域（x1,y1,x2,y2,...） -->
  <area shape="poly"
        coords="400,100,450,150,420,200"
        href="nagoya.html"
        alt="名古屋">
</map>
shape属性の値coords属性の形式説明
rect左上x, 左上y, 右下x, 右下y矩形（長方形）
circle中心x, 中心y, 半径円形
polyx1, y1, x2, y2, ... （3点以上）多角形
default不要画像全体

⑧ object要素

ポイント
・<object> は外部リソース（PDF・画像・動画など）を埋め込む要素です。
・embed 要素と異なり、フォールバックコンテンツを子要素として記述できます。
・data 属性でリソースのURLを、type 属性でMIMEタイプを指定します。 
<!-- PDFの埋め込み -->
<object data="document.pdf"
        type="application/pdf"
        width="800"
        height="600">
  <!-- フォールバック：表示できない場合に表示される -->
  <p>PDFを表示できません。
    <a href="document.pdf">こちらからダウンロード</a>
  </p>
</object>

<!-- 画像の埋め込み（imgのフォールバックとして） -->
<object data="photo.jpg"
        type="image/jpeg"
        width="400"
        height="300">
  画像を表示できません。
</object>
属性説明
data埋め込むリソースのURL
typeMIMEタイプ
width / height表示サイズ（ピクセル整数値のみ）
nameフォーム送信時の名前
form関連するフォームのID
✅ embed と object の違い
・embed：空要素。フォールバックコンテンツを持てない
・object：フォールバックコンテンツを子要素として持てる

🖊️ 演習問題（入力して答え合わせ）

入力欄に答えを入れて「答え合わせ」ボタンを押してください。

Q1. 正誤問題（○か×を入力）

<img> 要素の width 属性には "50%" のようなパーセント値を指定できる。


Q2. 選択問題

<picture> 要素内の <source> 要素について、 正しい説明はどれですか？


Q3. 穴埋め問題（入力）

以下の説明に合う属性名または要素名を入力してください。


Q4. 正誤チェックボックス問題

正しい記述をすべて選んでください。


Q5. コード穴埋め問題

以下のコードの __A__ 〜 __E__ に適切な属性名を入力してください。

<picture>
  <source __A__="(min-width: 768px)"
          __B__="photo-large.jpg">
  <img src="photo-small.jpg" __C__="風景写真">
</picture>

<video controls __D__="thumbnail.jpg">
  <source __E__="movie.mp4" type="video/mp4">
</video>

Q6. 記述問題

alt 属性を省略することと alt=""（空文字）を指定することの違いを説明してください。 また、alt 属性を省略できる条件を述べてください。


📌 要素まとめ一覧

要素役割重要ポイント
img 画像の埋め込み（空要素） alt省略は特別なケースのみ可。width/heightはピクセル整数値のみ（%不可）
picture 複数画像候補から最適なものを表示 必ずimgフォールバックを含める。source要素のsrcsetが必須
source メディアの候補ソースを指定（空要素） picture内：srcset必須 / video・audio内：src必須
video 動画の埋め込み controls / autoplay / muted / loop / poster属性。source要素で複数フォーマット指定可
audio 音声の埋め込み videoとほぼ同じ構造。width/height属性なし
track 字幕・キャプション・チャプター追加（空要素） video・audio両方で使用可。kind属性で種類指定。WebVTT形式
embed 外部コンテンツの埋め込み（空要素） フォールバックコンテンツを持てない
map イメージマップの定義 imgのusemap属性でname属性を参照
area イメージマップのクリック領域（空要素） shape属性：rect / circle / poly / default
object 外部リソースの埋め込み フォールバックコンテンツを子要素として持てる（embedとの違い）
