🏗️ HTML5 全体構造・メタ情報

html / head / body / title / meta / link / base 要素の役割と使い方を学びましょう。

① html要素

ポイント
・<html> はHTML文書全体を包むルート（根）要素です。
・すべての要素はこの要素の子孫になります。
・lang 属性で文書の言語を指定します（アクセシビリティ・SEOに重要）。
・開始タグ・終了タグともに省略可能ですが、明示的に書くことが推奨されます。 
<!-- lang属性で言語を指定 -->
<html lang="ja">          <!-- 日本語 -->
<html lang="en">          <!-- 英語 -->
<html lang="zh-CN">       <!-- 中国語（簡体字） -->

<!-- XHTMLの場合はxmlns属性も必要だった（HTML5では不要） -->
<html xmlns="http://www.w3.org/1999/xhtml" lang="ja">
属性説明例
lang 文書の言語を指定（BCP 47言語タグ） ja, en, fr
dir テキストの方向（ltr: 左→右 / rtl: 右→左） ltr, rtl
⚠️ lang属性を省略すると…
スクリーンリーダーが誤った言語で読み上げる場合があります。
SEOにも影響するため、必ず指定しましょう。

② head要素

ポイント
・<head> はブラウザ向けのメタ情報を格納するセクションです。
・ページには表示されません（タブタイトル・スタイル・スクリプトなど）。
・開始タグ・終了タグともに省略可能ですが、明示的に書くことが推奨されます。
・直下に置ける要素：title / meta / link / style / script / base など 
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>ページタイトル</title>
  <link rel="stylesheet" href="style.css">
  <script src="app.js" defer></script>
</head>

③ body要素

ポイント
・<body> はブラウザに実際に表示されるコンテンツを格納します。
・1つのHTML文書にbody要素は1つだけです。
・開始タグ・終了タグともに省略可能ですが、明示的に書くことが推奨されます。 
<body>
  <header>ヘッダー</header>
  <main>
    <h1>メインコンテンツ</h1>
    <p>本文テキスト</p>
  </main>
  <footer>フッター</footer>
</body>

④ title要素

ポイント
・<title> はブラウザのタブや検索結果に表示される文書タイトルを定義します。
・<head> 内に記述します。
・タグや文字参照以外のテキストのみを含めることができます（子要素は持てません）。 
<title>HTML5 学習サンプル | MeshinoKiwamiOkome</title>

<!-- ❌ タグは含められない -->
<title><strong>太字タイトル</strong></title>  <!-- 不正 -->
⚠️ 試験重要ポイント：title要素の省略

HTML5よりも前（HTML4 / XHTML）：
　head要素内に title要素を必ず1つ記述する必要があった。

HTML5以降：
　以下の条件を満たす場合に限り、title要素を省略できる。

　✅ より上位のプロトコルがタイトル情報を提供している場合
　　（例：メールのSubjectヘッダーがHTMLメールのタイトルを提供する場合）

　❌ 通常のWebページでは省略すべきではない
　（SEO・アクセシビリティ・ユーザビリティの観点から必須）
✅ titleのベストプラクティス
・ページの内容を端的に表すタイトルにする
・「ページ名 | サイト名」の形式が一般的
・全角30〜35文字程度が検索結果で表示される目安

⑤ meta要素

ポイント
・<meta> はページに関するメタデータを定義する空要素です。
・<head> 内に記述します。
・用途によって使う属性の組み合わせが変わります。

charset属性（文字コード指定）

⚠️ 試験重要ポイント：charset属性に指定できる値

HTML5では charset 属性に指定できる値は UTF-8 のみです。

✅ <meta charset="UTF-8">　← 正しい
❌ <meta charset="Shift_JIS">　← HTML5では不正
❌ <meta charset="EUC-JP">　← HTML5では不正

※ HTML5以前では Shift_JIS や EUC-JP も使われていましたが、
　HTML5仕様では UTF-8 のみが適合する文字エンコーディングとして定義されています。

また、charset 属性を持つ meta 要素は 文書内に1つだけしか置けません。 
<!-- 文字コード指定（必ずhead内の先頭付近に記述） -->
<meta charset="UTF-8">

name / content属性（メタデータ指定）

<!-- ページの説明（検索結果のスニペットに使われる） -->
<meta name="description"
      content="HTML5プロフェッショナル認定試験レベル1の学習サンプルです。">

<!-- キーワード（現在は検索エンジンにほぼ無視される） -->
<meta name="keywords" content="HTML5, 学習, 試験対策">

<!-- 作者情報 -->
<meta name="author" content="MeshinoKiwamiOkome">

<!-- レスポンシブデザイン用ビューポート設定 -->
<meta name="viewport"
      content="width=device-width, initial-scale=1.0">

<!-- 検索エンジンへの指示 -->
<meta name="robots" content="noindex, nofollow">

http-equiv属性（HTTPヘッダー相当の指定）

<!-- ページの自動リダイレクト（5秒後にURLへ移動） -->
<meta http-equiv="refresh" content="5; url=https://example.com">

<!-- IEの互換モード指定（レガシー対応） -->
<meta http-equiv="X-UA-Compatible" content="IE=edge">

<!-- コンテンツセキュリティポリシー -->
<meta http-equiv="Content-Security-Policy"
      content="default-src 'self'">
属性の組み合わせ用途
charset 文字エンコーディング指定（UTF-8のみ）
name + content ページのメタデータ（description / keywords / viewport など）
http-equiv + content HTTPヘッダー相当の指示（refresh / X-UA-Compatible など）
property + content OGP（Open Graph Protocol）などの拡張メタデータ

⑥ link要素

ポイント
・<link> は外部リソースとの関係を定義する空要素です。
・主にCSSファイルの読み込みやファビコン指定に使います。
・rel 属性（関係の種類）と href 属性（URL）が基本セットです。
⚠️ 試験重要ポイント：type属性のデフォルト値

rel="stylesheet" でスタイルシートを読み込む際、
type 属性のデフォルト値は text/css です。

そのため、CSSファイルを読み込む場合は type 属性を省略できます。

✅ <link rel="stylesheet" href="style.css">　← 省略OK（推奨）
✅ <link rel="stylesheet" type="text/css" href="style.css">　← 明示もOK
❌ <link rel="stylesheet" type="text/javascript" href="style.css">　← 誤り 
<!-- CSSファイルの読み込み（type属性省略・推奨） -->
<link rel="stylesheet" href="style.css">

<!-- type属性を明示する場合 -->
<link rel="stylesheet" type="text/css" href="style.css">

<!-- ファビコン（ブラウザタブのアイコン） -->
<link rel="icon" href="favicon.ico">
<link rel="icon" type="image/png" href="favicon.png">

<!-- 正規URL（SEO対策：重複コンテンツの統一） -->
<link rel="canonical" href="https://example.com/page">

<!-- フォントの先読み（パフォーマンス最適化） -->
<link rel="preload" href="font.woff2" as="font" crossorigin>

<!-- 代替スタイルシート -->
<link rel="alternate stylesheet" href="dark.css" title="ダークモード">

<!-- メディアクエリと組み合わせ（印刷用CSS） -->
<link rel="stylesheet" href="print.css" media="print">
rel属性の値用途
stylesheet外部CSSファイルの読み込み
iconファビコン（タブアイコン）の指定
canonical正規URLの指定（SEO）
preloadリソースの先読み（パフォーマンス）
alternate代替ページ（別言語版・RSSフィードなど）
author作者情報ページへのリンク

⑦ base要素

ポイント
・<base> は文書内のすべての相対URLの基準となるURLを指定する空要素です。
・<head> 内に記述し、文書内に1つだけしか置けません。
・href 属性（基準URL）と target 属性（リンクのデフォルト開き方）を指定できます。 
<!-- base要素の基本 -->
<head>
  <base href="https://example.com/docs/" target="_blank">
</head>

<body>
  <!-- 相対URLは base href を基準に解決される -->
  <a href="guide.html">ガイド</a>
  <!-- 実際のURL: https://example.com/docs/guide.html -->

  <img src="images/logo.png" alt="ロゴ">
  <!-- 実際のURL: https://example.com/docs/images/logo.png -->
</body>
属性説明値の例
href 相対URLの基準となるURL https://example.com/docs/
target リンクのデフォルトの開き方 _blank, _self, _parent, _top
⚠️ base要素の注意点
・文書内に1つだけしか置けません（2つ目以降は無視される）
・href または target の少なくとも一方を指定する必要があります
・ページ内リンク（href="#section"）も base href を基準に解決されるため、
　base要素を使うとページ内リンクが意図通りに動作しなくなる場合があります
